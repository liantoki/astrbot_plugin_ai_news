# astrbot_plugin_ai_news

<p align="center">
  <img src="logo.png" alt="astrbot_plugin_ai_news logo" width="160">
</p>

AstrBot AI 新闻整合插件。插件优先从固定 RSS/Atom/JSON Feed 信息源抓取新闻，在候选不足时使用搜索 API 补充，再进行去重、评分、AI 改写整合，并输出图片新闻卡片或文字消息。

当前版本：`1.1.1`

## 功能

- 固定信息源优先：默认内置多个 RSS/Atom 源，并支持在配置页增删改。
- 搜索补充：固定源候选不足时，可使用 Tavily、Bocha、Baidu 或 AstrBot 搜索工具补充。
- 多领域整合：支持配置默认新闻领域，也支持 `/ainews 人工智能,半导体` 临时指定。
- 领域别名归一：例如 `AI`、`AIGC`、`大模型` 可匹配 `人工智能` 信息源。
- 去重与评分：按 URL、标题相似度、来源权重、时效和内容质量排序。
- 低价值内容降权/过滤：对周报、盘点、论坛、培训、方案、白皮书等低新闻性内容进行处理。
- 分批 AI 改写：先改写高分候选，不足再补下一批，降低生成时间和 token 消耗。
- 定时推送：支持订阅每日固定时间推送。
- 图片卡片：优先输出图片卡片，失败时自动降级为带链接的文字新闻。

## 当前流程

```text
用户指定领域
-> 领域别名归一
-> 按领域筛选固定信息源
-> 并发抓取 RSS/Atom/JSON Feed
-> 时效过滤、相关性过滤、低价值内容过滤/降权
-> 候选不足时调用搜索 API 补充
-> 去重和评分排序
-> 分批调用 LLM 改写，数量不足则继续下一批
-> 改写后再次过滤和去重
-> 输出图片卡片或文字消息
```

## 依赖

```bash
pip install -r requirements.txt
```

`requirements.txt` 包含：

- `Pillow`：生成图片新闻卡片。
- `aiohttp`：抓取 RSS/Atom/JSON Feed、调用搜索 API、验证链接。

字体下载使用 Python 标准库 `urllib`，不额外依赖 `requests`。

## 配置说明

以下名称对应插件配置界面中的文字选项。

### 定时推送

- **推送小时**：每天定时推送的小时，范围 0-23。
- **推送分钟**：每天定时推送的分钟，范围 0-59。

### AI 整合设置

- **用于搜索整合的 AI 模型**：选择用于新闻改写和整合的模型。留空时使用当前会话默认模型。
- **新闻领域**：默认新闻领域，多个领域用逗号分隔。命令 `/ainews 人工智能,半导体` 会临时覆盖这里的设置。
- **每个领域新闻数量**：每个领域最多输出几条新闻。
- **候选新闻池倍数**：每个领域先保留多少倍候选进入排序和后续改写。数值越大越不容易缺新闻，但会增加后续处理成本。
- **优先生成图片卡片**：关闭后直接输出文字新闻；图片生成失败时也会自动降级为文字。

默认信息源建议使用这些领域名：

```text
人工智能、半导体、科技、数码、软件、硬件、安全、开源、创业、互联网、新能源、国际、中国、财经、金融、经济、商业、政治、社会
```

### 固定信息源

- **启用固定信息源**：开启后优先从固定 RSS/Atom/JSON Feed 抓取新闻。
- **固定信息源列表**：可在配置页增删改默认源，也可以添加自己的 RSS/Atom/JSON Feed。
- **固定源并发数**：同时抓取多少个固定源。源较多时可适当调高，但过高会增加网络压力。
- **新闻领域别名归一**：维护领域同义词，例如 `ai` 匹配 `人工智能`，`芯片` 匹配 `半导体`。
- **启用搜索 API 补充**：固定源候选不足时，是否继续调用搜索 API 或 AstrBot 搜索工具补充。
- **搜索补充前最少候选**：固定源候选少于该值时强制启动搜索补充。为 0 时使用自动判断。
- **搜索补充阈值**：固定源保留候选少于该值时才补搜，避免固定源已足够时还跑多轮搜索。
- **AI 改写额外候选数**：默认改写批次在最终需要数量之外多带几条候选，降低二次过滤后数量不足的概率。
- **AI 改写批次大小**：每次送给 LLM 改写多少条候选新闻。
- **AI 改写并发批次**：同时运行多少个 LLM 改写批次。提高可降低等待时间，但可能触发限流或增加瞬时成本。

固定信息源中的每一项包含：

- **来源名称**：卡片中显示的来源名。
- **订阅地址**：RSS/Atom/JSON Feed 地址。
- **适用领域**：逗号分隔。留空表示所有领域都使用该源。
- **权重**：数值越高越优先保留该来源的候选新闻。

订阅地址支持占位符：

```text
{topic}      URL 编码后的当前领域
{topic_raw}  原始当前领域
{days}       新闻最大时效对应的天数
{hours}      新闻最大时效对应的小时数
```

### Google 新闻搜索源注意事项

默认源中的 **Google新闻搜索** 是聚合搜索源，不是原始媒体 RSS。它适合作为全领域兜底补充，但不适合作为主要信息源。建议保持较低权重。

插件会对 Google 新闻搜索返回的周报、盘点、方案、培训、论坛、研讨会、合作共建等低新闻性结果进行过滤或降权。

### 搜索工具

- **启用的搜索函数工具**：按顺序尝试 AstrBot 已配置的搜索工具。
- **直连搜索 API**：插件自己直接请求的搜索 API，可选 Tavily、Bocha、Baidu。
- **Tavily API Key**、**Bocha API Key**、**Baidu AppBuilder API Key**：可在插件中填写；留空时会尝试从 AstrBot 配置文件中读取。
- **使用 AstrBot 搜索工具**：关闭后不再调用 AstrBot 的 `web_search_*` 工具。
- **最大搜索轮次**：搜索补充时最多换关键词搜索多少轮。
- **新闻最大时效（天）**：只保留最近多少天内发布或发生的新闻。
- **并行搜索轮次**：多轮搜索时同时执行多少轮。
- **搜索工具失败后停止**：某个搜索工具失败后是否停止继续尝试后续工具。

### 推送与超时

- **推送目标**：订阅推送的会话目标，通常由 `/ainews_sub` 自动维护。群聊和私聊是不同会话，建议分别在目标群聊或私聊中执行 `/ainews_sub`；手动填写时用逗号分隔，例如群聊 `aiocqhttp:GroupMessage:123456789`，私聊 `aiocqhttp:FriendMessage:987654321`。
- **搜索与验证超时**：抓取固定源、搜索 API 和链接验证的最长等待时间。

## 性能建议

通常耗时主要来自 LLM 改写，而不是 RSS 抓取。固定源抓取已经按 **固定源并发数** 并发执行。

想更快：

- 降低 **AI 改写批次大小**。
- 将 **AI 改写并发批次** 调到 2。
- 降低 **每个领域新闻数量**。
- 减少固定源中响应慢或质量低的源。

想更稳：

- 提高 **AI 改写额外候选数**。
- 提高 **AI 改写批次大小**。
- 保持较低的 **AI 改写并发批次**，减少限流风险。

## Docker 注意事项

如果 AstrBot 插件管理器没有自动安装依赖，需要进入容器执行：

```bash
pip install -r /AstrBot/data/plugins/astrbot_plugin_ai_news/requirements.txt
```

如果容器没有可用中文字体，可安装系统字体：

```bash
apt update && apt install -y fonts-noto-cjk
```

也可以把字体文件放到插件目录：

```text
astrbot_plugin_ai_news/fonts/NotoSansCJKsc-Regular.otf
```

## 字体策略

插件生成图片卡片时会按以下顺序查找中文字体：

1. 环境变量 `AI_NEWS_FONT_PATH` 指定的字体文件。
2. 插件目录 `fonts/` 下的 `.otf`、`.ttf`、`.ttc` 字体。
3. Linux、macOS、Windows 常见系统字体目录。
4. 自动下载开源 Noto CJK 字体到插件 `fonts/` 目录。

如果字体加载或图片生成失败，会自动降级为文字新闻。
