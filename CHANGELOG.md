# Changelog

## 1.1.1

### 新增

- 更换为更明亮的蓝色插件图标，并在 README 顶部展示图标。
- 配置页「推送目标」提示补充群聊和私聊 session ID 的区别与示例。

### 优化

- 多领域新闻收集改为并行执行，减少用户指定多个领域时的串行等待。
- 直连搜索 provider 改为并行请求，降低 Tavily、Bocha、Baidu 顺序等待造成的耗时。
- AI 改写批次使用独立 session ID，避免部分 provider 按同一会话串行处理并发请求。
- 删除重复的 `_translate_news_to_chinese` 实现，恢复带 `keep=false` 的候选过滤逻辑。

### 修复

- `/ainews_sub` 和 `/ainews_unsub` 现在写入 `push_settings.push_targets`，与配置界面字段保持一致。
- 清理未引用的 `formatter.py` 和未使用辅助方法。

## 1.1.0

### 为什么改

旧版本以搜索工具为主要新闻来源。实测中，免费新闻 API 和通用搜索结果经常存在实时性不足、重复率高、摘要像搜索结果而不像新闻、可用候选比例低的问题。1.1.0 将获取策略调整为固定信息源优先，搜索 API 只作为补充，以提高稳定性、速度和新闻质量。

### 新增

- 固定 RSS/Atom/JSON Feed 信息源优先抓取。
- 固定源结构化配置，支持在配置页增删改来源、URL、领域和权重。
- 默认信息源列表，覆盖 AI、科技、财经、国际、中文和英文技术资讯等常见方向。
- 领域别名归一配置，例如 `AI`、`AIGC`、`大模型` 可匹配 `人工智能`。
- Google 新闻搜索作为全领域兜底源，并在配置说明中标明其聚合搜索源属性。
- 固定源并发抓取。
- 低新闻性内容过滤/降权，包括周报、盘点、论坛、培训、方案、白皮书、共建等。
- LLM 分批改写机制：先处理高分候选，数量不足再补下一批。
- LLM 改写批次大小和并发批次数配置。
- `/ainews_reset_sources`：恢复默认固定信息源。
- `/ainews_reset_aliases`：恢复默认领域别名。
- `/ainews_diag` 输出固定源、别名、搜索 API 等更多诊断信息。

### 调整

- 搜索 API 从主流程改为候选不足时补充。
- Google 新闻搜索默认权重降低，避免压过真实 RSS 来源。
- 候选排序综合来源权重、时效、领域相关性、内容质量和重复度。
- 改写前不再一次性把全部候选交给 LLM，降低生成耗时和 token 消耗。
- 旧版 `name|url|topics|weight` 字符串列表会自动迁移为结构化固定源配置。

### 注意事项

- 如果配置页没有出现默认固定源，可执行 `/ainews_reset_sources`。
- 如果配置页没有出现默认领域别名，可执行 `/ainews_reset_aliases`。
- Google 新闻搜索是聚合搜索源，不是原始媒体 RSS，建议只作为兜底补充。
- 提高「AI 改写并发批次」可以降低等待时间，但可能触发模型服务限流或增加瞬时成本。
